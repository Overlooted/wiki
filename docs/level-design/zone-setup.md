# Zone Setup

## Scene Checklist

For each zone scene (auto-generated by `ZoneSceneBuilder`, but check after manual edits):

- [ ] `ZoneLoader` component present with `_zoneId` set (e.g. `zone.crypt`)
- [ ] Spawn point GameObjects named after `spawnPointId` values in `zones.json`
- [ ] Cart trigger collider tagged `"Cart"`
- [ ] Player tagged `"Player"`
- [ ] `GoblinSpawner` present with prefab + spawn/escape points assigned
- [ ] `PortalMono` on the portal wall with `_zoneId` and `_portalVisual` assigned
- [ ] `LandingPadMono` trigger near the portal

## zones.json

```json
{
  "id": "zone.crypt",
  "displayNameKey": "zone.crypt",

  ]
}
```

## Physics Layers

| Layer | Purpose |
|---|---|
| `Default` | Items on the ground |
| `InCartLoot` | Items inside the cart (index 6) |

**Layer Collision Matrix** (set in Project Settings → Physics):

- `InCartLoot` vs `InCartLoot` — **disabled** (items don't push each other inside cart)
- `InCartLoot` vs `Default` — **disabled**
- `InCartLoot` vs cart walls — **enabled**

## Tags

| Tag | Required by |
|---|---|
| `Player` | `ZonePortalMono`, `HumanoidLookAt` |
| `Cart` | `GoblinMono.OnTriggerEnter` |
| `Gem` | `GemSocketMono.CompareTag` |
| `Sword` | `CuttableCableMono.CompareTag` |
| `Heavy` | `WeightTriggerMono` |

All tags are registered in `ProjectSettings/TagManager.asset`.

## Adding a New Zone

1. Add entry to `zones.json` in Core
2. Run **Overlooted → Build → 5 - Sync StreamingAssets**
3. Run **Overlooted → Build → 3 - Zone Scenes** — the new scene is created automatically
4. Reposition spawn point GameObjects as needed
5. Build out the level geometry and add portals manually
