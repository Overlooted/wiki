# Cart System

## CartMono

SpringJoint-based cart. Items are placed inside via GrabMaster. Calls `Cart.TryAddLoose()` on the Core model when an item enters the trigger zone.

Physics layers are critical — see [Physics Layers](../level-design/zone-setup.md#physics-layers).

## CrateMono

Seals loose items inside the cart permanently. On `Seal()`:

1. Calls `Cart.Seal()` on the Core model
2. Calls `ItemMono.DestroyPhysics()` on all contained items (removes `Rigidbody`)
3. Locks the crate lid

Sealed items cannot be stolen by goblins.

## WheelControl

Reads `WheelCollider` physics each frame and rotates the visual wheel mesh to match. No additional setup needed beyond assigning the collider and mesh transform references.

## BrakeLever

Interactive brake. Rotates a lever transform and sets `WheelCollider.brakeTorque`. Player interacts with it using the interact key.

## HandleMover

Trigger-based handle repositioning. The player can move the cart handle between front and rear positions by walking into the trigger zone.

## WheelSqueak

Plays audio at a pitch proportional to wheel RPM. Assign the `AudioSource` and set `minRpm` / `maxRpm` pitch range in the inspector.

## CartRecaller

`Interactable` that teleports the cart back to its home position (a child GameObject named `CartHomePosition`). Place in the hub scene near the portal so the player can retrieve a runaway cart.

## Cart Scene Setup

The cart is auto-generated by `ZoneSceneBuilder`. It includes:

- `BoxCollider` (trigger) tagged `"Cart"` — required by `GoblinMono`
- `Rigidbody`
- `CartMono`
- `WheelControl`, `BrakeLever`, `HandleMover`
- World-space canvas showing current capacity
